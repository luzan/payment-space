# Payment Facilitator Platform Learning Guide

A 12-week structured learning plan for understanding payment facilitator platforms, designed for senior software engineers building fintech applications.

## Overview

This guide covers the complete knowledge base needed to build and operate a payment facilitator (PayFac) platform. The learning is organized into six two-week modules, each building upon the previous.

## Learning Modules

| Week  | Module                                                                  | Focus Area                                      |
| ----- | ----------------------------------------------------------------------- | ----------------------------------------------- |
| 1-2   | [Payment Ecosystem Foundation](./week-01-02-payment-ecosystem/)         | Core entities, money flow, industry players     |
| 3-4   | [Merchant Onboarding & Underwriting](./week-03-04-merchant-onboarding/) | KYC/KYB, risk assessment, compliance collection |
| 5-6   | [Risk & Compliance](./week-05-06-risk-compliance/)                      | Chargebacks, fraud, PCI-DSS, AML/BSA            |
| 7-8   | [Transaction Processing](./week-07-08-transaction-processing/)          | Authorization, settlement, message formats      |
| 9-10  | [Platform Architecture](./week-09-10-platform-architecture/)            | Data modeling, audit trails, event systems      |
| 11-12 | [Regulatory & Partnerships](./week-11-12-regulatory-partnerships/)      | Sponsor banks, card network registration, MTLs  |

## How to Use This Guide

Each module contains:

- **README.md** - Overview and learning objectives
- **topics.md** - Detailed topics to research
- **questions.md** - Key questions to be able to answer after completing the module
- **resources.md** - Suggested reading and research sources

## Recommended Approach

1. Read each module's README to understand the scope
2. Work through the topics systematically
3. Research each topic using official documentation and industry resources
4. Validate understanding by answering the module's key questions
5. Build mental models by mapping concepts to real-world payment flows

## Prerequisites

- Understanding of distributed systems architecture
- Familiarity with API design and integration patterns
- Basic knowledge of financial terminology
